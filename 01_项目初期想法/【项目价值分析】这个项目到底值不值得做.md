# 项目价值分析：这个项目到底值不值得做？

> **问题**：我的项目都是之前的业务经验，逐步落地后用在面试会怎么样？面试官会怎么看我？有没有必要做？
> **结论**：必须做，但要做对。蓝图展示视野，MVP展示深度。

---

## 一、项目评分：8.5/10（培训班项目中Top 5%）

| 维度 | 评分 | 理由 |
|------|------|------|
| **业务真实性** | ⭐⭐⭐⭐⭐ | 不是编造的场景，是你在比亚迪+瑞能亲历的 |
| **差异化** | ⭐⭐⭐⭐⭐ | 培训班99%的人做聊天机器人/通用问答，你做汽车CRM全链路 |
| **竞品分析深度** | ⭐⭐⭐⭐⭐ | 能逐条拆解纷享销客6大能力，说明你不是闭门造车 |
| **技术栈覆盖** | ⭐⭐⭐⭐ | RAG+Agent+图谱+Text-to-SQL+多模态，覆盖面广 |
| **可行性** | ⭐⭐⭐ | 48个模块蓝图太庞大，MVP能否聚焦落地是最大风险 |
| **面试说服力** | ⭐⭐⭐⭐ | 有真实业务底座，但要能讲清楚"我做出来的部分" |

### 三个最大优势

```
1. 真实业务锚点 → 你不是在"想象"痛点，你"活过"这些痛点
2. 行业纵深 → 汽车+制造+ToB CRM，这个交叉领域人才极度稀缺
3. 系统级思维 → 48个模块的全链路规划，展示的是产品架构能力
```

### 三个最大风险

```
1. 蓝图太大做不完 → 面试官问"你实际做了多少"，你说5/48，印象打折
2. 容易变成"PPT工程师" → 规划了但没落地，面试官觉得你只会画饼
3. 技术深度不够 → RAG召回率多少？Agent怎么编排？幻觉怎么控？答不上来就翻车
```

## 二、面试官会怎么看你？

### 正面画像（做好时）

| 面试官评价 | 潜台词 |
|-----------|--------|
| "这个人有业务sense" | 不像纯技术人只懂调API |
| "他在原行业积累了真实经验" | 转行不是从零开始 |
| "他对CRM的理解比我见过的大多数候选人都深" | 全链路蓝图展示了视野 |
| "他能做竞品分析，有产品思维" | 拆解纷享销客说明你会研究市场 |
| "行业复合型人才，可塑性很强" | 汽车+AI，市场上太少见了 |

### 负面风险（必须避免）

| 面试官质疑 | 触发条件 | 如何规避 |
|-----------|---------|---------|
| "规划了很多，但实际做了多少？" | 只能演示2-3个浅层功能 | **MVP 5个功能必须做深做透**，能现场演示 |
| "是不是只停留在概念层面？" | 讲不清RAG召回率、Agent状态机细节 | 每个核心功能必须能讲出**3层技术细节** |
| "功能太多太散，没有聚焦" | 蓝图48个功能但每个都很浅 | 面试时主动说"蓝图展示视野，MVP聚焦5个做深" |
| "这个项目能上生产吗？" | 说能，被追问就露馅 | 诚实说"是验证架构的MVP，生产化需要..." |
| "转行的，技术功底够吗？" | LeetCode做不出、基础概念答不上来 | 基础知识不能丢，项目是加分项不是替代品 |

## 三、面试官认为你是什么样的人？

### 如果做好了 →

```
"这是一个懂业务、有产品思维、能用AI技术解决真实问题的
 行业复合型人才。他不是纯技术出身但技术能力在进步，
 他的价值在于能把AI技术落地到具体行业场景中。"

 → 适合岗位：AI产品经理 / AI应用开发 / 行业解决方案
 → 不适合岗位：核心算法研究 / 底层框架开发
```

### 如果没做好 →

```
"这个人业务理解还可以，但技术落地能力不足，
 规划了很多但实现太少，可能是执行力的问题。"

 → 结论：理论型选手，不敢给offer
```

**区别只在一件事：MVP有没有做出来、做深了。**

## 四、有没有必要做这个项目？

### 三种选择对比

| 不做这个项目 | 做了但做错了 | 做了且做对了 |
|------------|------------|------------|
| 面试只有培训班通用项目 | 蓝图很美但做不出来 | 蓝图展示视野+MVP展示深度 |
| 和其他同学同质化竞争 | 面试官觉得只会画饼 | 面试官觉得既懂业务又能落地 |
| 无法展示差异化 | 比没做还差 | **Top 5%的候选人** |

### "做对了"的最低标准

```
✅ 5个MVP功能全部跑通，能现场演示
✅ 每个功能能讲清3层技术细节（为什么选这个技术→怎么实现的→效果怎么样）
✅ GitHub有完整代码 + README + 演示视频
```

### "做对了"的加分标准

```
🎯 RAG召回率 > 70%，有测试数据证明
🎯 Agent状态机图能画出来，讲清路由逻辑
🎯 有1-2个"我踩过的坑和怎么解决的"故事
🎯 能讲出和纷享销客的3个差异化设计思路
```

## 六、面试话术模板

### 讲项目时的标准结构（STAR + 差异化）

```
Situation（背景）：
"我在比亚迪做售后、在瑞能用纷享销客做售前，亲历了CRM系统的
 6个核心痛点——故障信息碎片化、保修审核慢、销售抵触录入..."

Task（目标）：
"所以我决定用AI重构整个CRM流程，做了一个汽车行业专属的
 AI CRM智能助手。"

Action（行动 - 讲3层）：
"我详细研究过纷享销客刚发布的AI销售助理6大能力，并在此基础上
 做了3个差异化：
 1. 行业纵深——深耕汽车售后（故障诊断RAG、保修索赔Agent）
 2. AI深度——不只语音转文字，还做了RAG推理+知识图谱+多Agent协作
 3. 全链路——覆盖售前→售中→售后→治理的完整闭环

 MVP阶段我聚焦5个核心功能做深：
 - 故障诊断RAG（召回率XX%）
 - 对话信息抽取 + CRM自动更新
 - 语言对话创建工单
 - 智能问数（Text-to-SQL）
 - 统一Web界面"

Result（结果）：
"诊断时间从模拟15分钟降到3分钟，信息录入从30分钟降到5分钟。
 项目GitHub地址XXX，可以现场演示。"
```

### 应对质疑的话术

| 质疑 | 回答 |
|------|------|
| "你实际做了多少功能？" | "蓝图规划了48个模块展示系统思维，但MVP聚焦5个核心功能做深做透，每个都能现场演示" |
| "和商业产品差距大吗？" | "我详细拆解过纷享销客AI销售助理，我的差异化在行业纵深、AI深度和全链路覆盖，不是为了替代商业产品，而是验证架构设计能力" |
| "数据是真实的吗？" | "基于脱敏的模拟数据，但数据结构完全来自我亲历的真实业务流程，字段、关系、逻辑都是真实的" |
| "能上生产吗？" | "当前是MVP验证阶段，生产化需要考虑并发、缓存、分库分表、权限体系等，我在架构设计时已经预留了这些扩展点" |
