## 二、核心问题：我们在解决什么（Why）

### 2.1 行业痛点（来自真实工作 + CRM方案文档验证）

> **说明**：以下痛点来自**真实工作经历**，并经**瑞能股份×纷享销客CRM解决方案(0322/0428版)**和**泛微九氚汇CRM解决方案**两份企业级方案文档交叉验证。每一条都有真实的业务场景支撑。

| 痛点 | 来源 | 具体表现 | 造成的后果 | 方案文档出处 |
|------|------|----------|-----------|-------------|
| **售后故障信息碎片化** | 比亚迪售后系统 | 故障描述分散在工单、电话、聊天记录中，技师凭经验诊断 | 诊断耗时长（平均15分钟+）、误诊率高、客户满意度低 | 瑞能方案-ITR服务管理 |
| **保修索赔人工审核慢** | 比亚迪售后流程 | 保修政策条款复杂，人工匹配易出错、易遗漏 | 审核周期长、合规风险、客户抱怨 | 瑞能方案-保内保外区分 |
| **销售记录利用不足** | 瑞能纷享销客售前 | 历史销售数据、客户对话沉睡在系统中 | missed opportunities，新人上手慢 | 瑞能方案-LTC商机经营 |
| **商机推进靠感觉** | 瑞能纷享销客使用 | 销售凭直觉判断商机阶段，缺乏数据支撑 | 赢率预测不准、资源分配不合理 | 瑞能方案-毛利测算+阶段管理 |
| **联系人关系不透明** | 瑞能纷享销客 | 客户内部决策链不清楚，找不到关键影响人 | 跟单盲目、关键节点卡壳 | 瑞能方案-决策链识别 |
| **日报录入费时** | 比亚迪+瑞能 | 销售/技师每天花30分钟+写日报 | 低价值劳动、信息滞后 | 泛微方案-组织赋能 |
| **CRM操作繁琐，销售抵触** | 纷享销客一线反馈 | 查客户、建商机反复跳转页面，每天2-3小时耗在系统操作 | 销售敷衍录入→数据失真→管理决策失效 | 瑞能方案-业务流程优化 |
| **拜访记录补作业** | 纷享销客一线反馈 | 白天跑客户晚上补记录，信息大量遗漏 | 客户需求丢失、跟进断链 | 瑞能方案-拜访管理 |
| **客户情报搜集耗时** | 纷享销客一线反馈 | 拜访前花数小时零散搜集工商/财报/竞品信息 | 信息分散、难以形成有效业务判断 | 瑞能方案-客户360视图 |
| **新人上手慢、跨部门信息差大** | 纷享销客一线反馈 | 新人接手老客户需数天整理历史记录 | 信息孤岛、协同效率低 | 泛微方案-E-learning学堂 |
| **手动录单易出错** | 纷享销客一线反馈 | 合同签订后手动填数十个订单字段 | 审批卡壳、交付风险、回款周期拉长 | 瑞能方案-选型配置 |
| **周/日报无效复盘** | 纷享销客一线反馈 | 销售堆砌流水账，管理者汇总无效数据 | 找不到核心问题与增长机会 | 瑞能方案-BI数据分析 |
| **⭐客户报备查重难** | ⭐瑞能方案新增 | 模糊匹配和精准查重难，容易撞单抢单 | 优质客户重复跟进，内部恶性竞争 | 瑞能0428-P3 |
| **⭐快速报价选配难** | ⭐瑞能方案新增 | 价格控制和报价审核耗时耗力，参数配置错误率高 | 报价效率低，毛利核算不准 | 瑞能0428-LTC报价 |
| **⭐数据链割裂** | ⭐瑞能方案新增 | 售前/交付/服务数据未拉通 | 不清楚安装进度及款项回收情况 | 瑞能0428-数据闭环 |
| **⭐应收款管理难** | ⭐瑞能方案新增 | 回款计划不清晰，交付进度无法匹配应收款项 | 资金周转压力，逾期风险高 | 瑞能0428-LTC回款 |
| **⭐营销效果无法量化** | ⭐泛微方案新增 | 市场活动投入产出比ROI无法追踪 | 营销预算分配不合理，渠道效果不明 | 泛微方案-效果验证器 |

> **带⭐的6条痛点为从CRM方案文档中补充的新痛点（共17条），进一步完善了项目的业务覆盖面。**

### 2.2 2026年市场机会

- **AI CRM爆发**：纷享销客ShareAI、销售易NeoAgent等主流产品大力推AI Agent、RAG知识库、智能推进
- **汽车售后智能化**：新能源车三电系统复杂，AI辅助诊断需求强烈
- **企业缺复合型人才**：懂业务 + 能落地AI的人供不应求

### 2.3 核心问题总结

```
传统CRM是"被动记录系统"——人录入、人查询、人判断。
我们要把它变成"主动智能助手"——AI理解、AI建议、AI执行。
```

## 三、技术标准与演进原则（Tech Stack Rules）

> **核心原则**：用**最新且成熟**的技术，在自己的**能力边界内**做出**可运行、可演示、可讲清**的项目。不追噱头，不堆概念，每个技术选型必须有明确理由。

### 3.1 技术选型原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **最新但成熟** | 选2025-2026年主流、有社区支持的技术，不选实验性/文档残缺的新玩具 | LangGraph ✅（已成熟）、某论文刚发布的未开源框架 ❌ |
| **能力边界内** | 能在1-2周内上手并产出，不选需要3个月才能跑通的技术 | Chroma ✅（5分钟安装）、自研向量引擎 ❌ |
| **面试友好** | 选面试官认识、能展开讨论的技术，不选过于小众的工具 | LangChain/LangGraph ✅（大厂JD高频）、某国产小众框架 ❌ |
| **可替换性** | 每个模块设计为可插拔，后续发现更好的方案能平滑迁移 | 向量库Chroma ↔ Milvus、LLM通义 ↔ DeepSeek |
| **记录演进** | 每引入一项新技术，记录"为什么选它、替代方案是什么、什么情况下会换" | 见下方"技术演进日志" |

### 3.2 当前技术栈（v1.0）

| 层级 | 技术 | 选型理由 | 替代方案 | 升级触发条件 |
|------|------|---------|---------|-------------|
| **LLM** | 通义千问 / DeepSeek API | 国产、成本低、文档全、国内访问稳定 | 文心一言、GPT-4o | 课程要求/性能不足/成本变化 |
| **Agent框架** | LangGraph 0.3+ | 2026年最热门的Agent编排框架，状态机可视化，面试必问 | LangChain Expression、AutoGen | 课程更新/社区迁移 |
| **向量库** | Chroma 0.6+ | 轻量、本地可运行、5分钟上手 | Milvus、Qdrant、FAISS | 数据量>10万/需分布式 |
| **知识图谱** | NetworkX + PyVis | 零部署、Python原生、可视化效果好 | Neo4j、Dgraph | 节点>1万/需持久化 |
| **前端Demo** | Streamlit 1.40+ | 2小时出界面、纯Python、面试可现场改 | Gradio、React+FastAPI | 需要更复杂交互 |
| **后端接口** | FastAPI 0.115+ | 异步、自动生成文档、面试高频 | Flask、Django | 需要重业务逻辑 |
| **数据存储** | SQLite / JSON文件 | 零配置、Demo阶段够用 | PostgreSQL、MongoDB | 多用户并发/复杂查询 |

### 3.3 技术演进日志（持续更新）

| 日期 | 技术变动 | 原因 | 决策人 |
|------|---------|------|--------|
| 2026-04-26 | 初版技术栈确定 | 基于当前课程进度 + 市场热度 | 邬思彬 |
| | | | |
| | | | |

> **如何更新**：每当看到新的技术（新框架、新模型、新方法），先问自己三个问题：
> 1. 它比现在的方案好在哪里？（具体优势）
> 2. 我能在1周内上手并替换吗？（能力边界）
> 3. 面试官会认可这个选择吗？（面试价值）
> 三个都是Yes → 加入演进日志 → 在下一个里程碑尝试替换。

### 3.4 能力边界声明（诚实面对）

```
当前能力范围：
✅ Python基础 + PyTorch基础
✅ Prompt Engineering
✅ RAG概念理解
🔄 正在学：LangChain / LangGraph
❌ 尚未学：模型微调、分布式部署、前端开发

因此当前技术栈：
- 用API调用大模型（不调优模型本身）
- 用轻量级向量库（不自研检索引擎）
- 用Streamlit做界面（不写React）
- 用SQLite存数据（不搭数据库集群）

随着学习深入，逐步扩展边界，不急于求成。
```

| **FastAPI** | 后端API、异步处理 | 自动生成文档、异步高性能 | Flask、Django |
| **SQLite** | 轻量级业务数据存储 | 零配置、Demo阶段够用 | PostgreSQL（生产级）|

#### 流程A：售后故障诊断全流程

```
客户报修 → [交互层]语音/文字输入
  → [售后Agent] 信息抽取(客户/车型/症状)
  → [RAG] 检索故障案例库 + 维修手册
  → [知识图谱] 查询保修状态 + 历史维修
  → [售后Agent] 推理诊断(可能原因 + 维修步骤 + 备件清单)
  → [Tool Calling] 创建维修工单 + 推荐技师
  → [报告Agent] 更新日报 + 故障率统计
  → [交互层] 返回诊断结果 + 工单号 + 预估费用
```

#### 流程B：售前商机推进全流程

```
销售录入线索 → [售前Agent] 线索评分(行业/规模/预算)
  → 市场活动 → [自动化Agent] 生成活动方案 + ROI预估
  → 商机推进 → [售前Agent] Next Best Action:
      立项 → 方案设计 → 方案偏离分析 → 招投标 → 报价 → 定制化 → 总结
  → [图谱Agent] 识别决策链 → 关键人推荐
  → 拜访 → [录音Agent] 录音总结需求 → 自动生成拜访报告
  → 合同 → [自动化Agent] 合同风险审查 → 自动化审批
  → 订单履约 → 安调 → 实施 → [报告Agent] ROI复盘
  → 回款 → [自动化Agent] 逾期预警 → 智能催收
```

#### 流程C：智能问数全流程

```
用户提问 "上季度华东区Top5客户销售额"
  → [问数Agent] 意图识别 + 实体抽取
  → [Text-to-SQL] Schema映射 → SQL生成
  → [规则引擎] SQL安全校验(只读/限表)
  → 执行查询 → 结果验证
  → [报告Agent] 图表生成 + 趋势解读
  → [交互层] 返回可视化图表 + 文字洞察
```
