### 核心设计理念：嵌入式AI vs 对话式AI（⭐面试必讲）

> 来自白皮书原话："**在合适的时间问对问题，才是关键。AI应用最大的挑战在于如何让不同水平的用户，在合适的时机提出高质量的问题。**"

| 维度 | 嵌入式AI（我们的主策略） | 对话式AI（辅助） |
|------|------------------------|-----------------|
| 触发方式 | **业务场景自动触发**，用户无需主动提问 | 用户主动提问为主 |
| 数据存储 | 结构化存储到CRM字段 | 灵活但非结构化 |
| 用户要求 | 低——系统引导操作 | 高——需知道怎么提问 |
| 适用场景 | 销售录入、商机推进、报价审批、工单处理 | 经营分析、知识问答 |
| 我们的实现 | LangGraph状态机 + Tool Calling + 自动触发规则 | Chat界面作为补充 |

> **面试话术**："我参考了纷享销客ShareAI的'嵌入式AI优先'设计理念——不是让销售去学怎么跟AI聊天，而是在他们工作的每个关键节点，AI主动给出建议。比如创建商机时自动推荐阶段，填写报价时自动校验逻辑。"

### 合规与安全体系（来自白皮书）

| 安全措施 | 说明 | 我们项目的对应实现 |
|----------|------|------------------|
| 私有部署大模型 | 支持接入企业自有LLM | Ollama本地部署 + 可选云端API |
| CRM数据权限一致 | AI数据权限与个人数据权限严格一致 | RBAC中间件 + Agent数据过滤 |
| 敏感数据Masking | 脱敏传输 | PII字段自动脱敏 |
| AI审计日志 | 支持脱敏/明文双模式 | Agent执行日志记录 |
| 零留存协议 | 商用大模型数据不留存 | 本地Ollama优先，云端选国内模型 |

## AI建设优先级原则（来自白皮书Page 9）

> 不是所有场景都值得做AI——这些原则决定了我们为什么**先做售后再做售前、先做RAG再做多模态**。

| 原则 | 白皮书建议 | 我们的执行 |
|------|-----------|-----------|
| **平台优先 > 深度优先 > 场景优先** | 先建平台能力，再深度优化单场景 | 先搭LangGraph架构(7月)，再逐场景调优 |
| **增效 > 降本** | 先让业务效率提升，再考虑节省成本 | 所有Agent定位为"提效工具"而非"替代人力" |
| **高频 > 低频** | 聚焦高频场景，低频场景ROI不高 | 售后诊断/线索分配(每天用) > 投标/培训(偶尔用) |
| **用户 > 管理员** | 先服务一线使用者，再服务管理者 | 先做销售/技师用的Agent，再做管理层看板 |
| **场景嵌入式 > 对话式** | 在合适时机主动触发优于等用户提问 | 嵌入式AI为主，Chat界面为辅 |

> **面试话术**："我参考了纷享销客的场景建设原则——'增效大于降本、高频大于低频、用户大于管理员'。所以我把售后智能诊断（每天技师都在用）放在第一个月做，而投标分析这种低频场景放到P2。"

### 线索查重规则

- **查重字段组合**：公司名称 + 手机号
- **查重时机**：新建线索保存时自动触发
- **查重范围**：全部线索 + 全部客户
- **处理方式**：发现重复 → 弹窗提示已有记录（显示负责人+最后跟进时间）
- **特殊配置**：每条线索可单独设置"不参与查重"（用于测试数据等）

### 客户归属争议申诉机制

```
销售A录入客户 → 系统检测到该客户已被销售B跟进
    ↓
弹窗提示："该客户已被B在跟进，是否申请获取权限？"
    ↓
A选择"申请" → 生成【客户负责人变更申请】单
    ↓
系统通知B → B同意/拒绝（OA审批流）
    ↓
同意 → 客户归属变更为A；拒绝 → A收到拒绝原因
```

### 天眼查自动回填

- **触发条件**：国内客户新建时，填写公司名称后自动触发
- **回填字段**：主营业务范围、注册时间、注册资本、法定代表人、统一社会信用代码
- **数据来源**：天眼查API / 企业工商信息接口
- **我们的简化实现**：模拟接口返回 + 手动补全入口

### 市场活动效果自动统计

- **统计维度**：活动带来的线索数量 → 转化的商机数 → 最终订单数量/金额
- **统计周期**：活动结束后7天内自动生成初步报告
- **我们的AI增强**：结合LTC AI场景清单#25"续费/二次销售触发"，形成**营销全链路归因**

### 8月补充：企业级PaaS能力对标（来自纷享销客/泛微方案）

> 面试时展示：不仅做了Demo，还理解企业级CRM平台的完整架构能力。

#### 四流合一（来自纷享销客PaaS平台）

| 流程类型 | 说明 | 我们的对应实现 |
|----------|------|---------------|
| **审批流** | 合同评审、报价审核、费用报销等需要多级审批 | LangGraph状态机 + Tool Calling自动提交审批 |
| **业务流BPM** | 立项→方案→报价→签约→交付→验收的全流程 | LangGraph阶段推进器 + 条件触发 |
| **工作流** | 任务分配、通知提醒、超时预警 | 自动化Agent + 定时任务调度 |
| **阶段推进器** | 商机从线索→立项→方案→谈判→成交的阶段管理 | 商机推进Agent + Next Best Action |

#### 多维度+多组织数据权限（来自纷享销客权限架构）

| 权限维度 | 说明 | 实现方式 |
|----------|------|---------|
| **组织维度** | 总部/子公司/部门/团队的数据隔离 | `tenant_id` + `org_id` 字段过滤 |
| **数据归属** | 私有/公开只读/公开读写/人员共享 | `owner_id` + 共享规则表 |
| **多维树** | 按地区/产品线/行业等多维度组织 | 自定义`dimension_tree`字段 |
| **角色控制** | 功能权限(能做什么) + 数据权限(能看到什么) | RBAC表 + 数据过滤器中间件 |

#### BI数据分析架构（来自纷享销客BI + 泛微九氚汇）

| BI能力 | 说明 | 我们的实现 |
|--------|------|-----------|
| **多维交叉分析** | 按部门/时间/产品/区域等维度自由组合分析 | Text-to-SQL + 动态查询生成 |
| **下钻查看明细** | 从汇总数据逐级深入到单条记录 | 分层聚合 + 点击展开 |
| **预聚合加速** | 常用指标预计算，避免实时全量扫描 | 缓存热门查询 + 定时预计算 |
| **数据大屏** | 经营动态可视化看板 | Streamlit仪表盘组件 |
| **效果验证器** | 营销活动→线索→商机→合同的全链路转化追踪 | 归因分析Agent |
